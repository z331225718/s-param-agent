📡 S-Parameter Agent

对话式 RF 工程助手 — 用自然语言替代传统 S 参数处理软件

快速开始

1安装依赖 
pip install flask scikit-rf plotly numpy
2配置 LLM 
cd scripts
cp config.example.json config.json
# 编辑 config.json，填入 api_key

支持 DeepSeek / OpenAI / Ollama / vLLM 等任意 OpenAI 兼容 API。

3启动服务 
cd scripts
python app.py

浏览器打开 http://localhost:5050

4加载 S 参数文件

两种方式：

  • 上传：拖拽 .sNp 文件到虚线框
  • 本地路径：在输入框输入服务器本地路径，如 /data/LNA.s2p，回车加载（绕过 IT 上传限制）

手动绘图

加载文件后：

  1. 点击左侧文件列表中的网络
  2. 在参数区搜索或选择 S 参数（如 S21）
  3. 点击 🔄 生成图表

图表为交互式 Plotly：hover 看数值、滚轮缩放、拖拽平移、双击重置。

Agent 对话

在对话框输入自然语言指令，Agent 自动生成并执行代码：

输入效果
画 S21 和 S11 的 dB 图双曲线幅度图
画 S11 Smith 圆图，2-4GHzSmith 图 + 频率截取
画 S21 相位，对数频率相位图 + log 轴
级联 LNA.s2p 和 BPF.s2p，画 S21级联后画图
只看 1-5GHz，导出 S21 为 CSV频率截取 + 导出
把 LNA BPF AMP 级联，画对数 Smith 图链式级联 + Smith 图

参数搜索

大端口文件（如 .s64p，4096 个 S 参数）：

  • 在搜索框输入 S42 即时过滤匹配参数
  • 快捷按钮：全部 / 反射 / 传输 / 端口1~8
  • 已选参数以芯片形式展示，点击删除

本地路径直读

💡 适用场景：IT 安全策略禁止浏览器上传文件、文件过大超时、已在服务器本地有文件。

在仪表盘上传区域下的输入框填写本地路径，回车即可。服务端直接 rf.Network(path) 解析，不走 HTTP 传输。

配置文件

{
  "llm": {
    "api_key": "sk-xxx",
    "base_url": "https://api.deepseek.com",
    "model": "deepseek-chat",
    "temperature": 0.0,
    "max_tokens": 1500,
    "timeout_sec": 60
  }
}

也可设环境变量 DEEPSEEK_API_KEY 或 OPENAI_API_KEY。

安全机制

  • AST 白名单：只允许 skrf/plotly/numpy 的 import，拦截 eval/os/subprocess
  • subprocess 沙箱：代码在独立进程中执行，30s 超时
  • 危险调用删除：fig.show() / fig.write_html() 在 AST 前正则移除
